# docs 目录说明

用途：存放本项目的文档（计划、架构、进度、参考等），用于避免“实现已变但文档仍停留在旧状态”。

## 命名与组织
- 文件命名建议包含主题与日期，例如：`VSCode语义Token配色插件-工作计划_YYYY-MM-DD_HH-MM-SS.md`
- 文档内容应以“可验证事实”为主：以仓库代码、命令输出、或可复现步骤为依据。

## 维护者
- 默认维护者：AI（Codex CLI 会话）。

## 何时需要更新
- README 或对外行为（命令、UI、配置写入策略、存储格式）发生变化时
- 里程碑状态改变（完成/部分完成/取消）时
- 发现“文档/实现不一致”时（以当前实现为准同步修正文档，必要时记录差异与后续计划）
