# 项目协作约定（必读）

目标：避免“代码已改但进度丢失/机器不同步”。

## 变更记录与同步（强制）
1) **每次修改仓库内容（代码/文档/配置）都必须同步进度文档**：
   - 追加记录到：`docs/progress/工作进度.md`
   - 每条记录至少包含：时间戳、做了什么、为何要做、如何验证（命令或手工步骤）
2) **每次修改完成后必须提交并推送到远端**：
   - `git add -A`
   - `git commit -m "<简短信息>"`
   - `git push`

## 写作规范
- 新增说明文字优先中文，简短可执行。
- 只写可验证事实；不确定内容必须标注“待确认”并给出最小验证步骤。
- 不要在文档或提交信息中包含敏感信息（token、密码、个人隐私）。
